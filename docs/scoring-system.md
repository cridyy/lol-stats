# 评分系统说明

本文档对应当前代码中的评分实现，主要来源：

- `src/scoring.ts`：单局百分制评分、单局标签、评分提示。
- `src/roleClassifier.ts`：根据英雄和装备判断玩家本局定位。
- `src/playerProfile.ts`：玩家画像、单英雄画像、综合标签。

评分系统不是 Riot 官方评分，也不是固定按英雄模板评分。它的核心目标是给海克斯大乱斗、极地大乱斗一类团战模式提供一个“电子斗蛐蛐”视角下的可解释分数。

## 总体流程

1. 从对局数据中计算基础指标。
2. 根据英雄标签和出装判断本局定位。
3. 根据定位进入不同评分公式。
4. 得到 0-100 的单局得分。
5. 根据得分和关键指标生成单局评价标签。
6. 多局聚合后生成玩家画像、单英雄画像和画像标签。

所有百分比在公式中都按 `0-1` 的小数计算，例如 `25% = 0.25`。

## 基础指标

记：

- `clamp01(x) = min(1, max(0, x))`
- `ratio(a, b) = b > 0 ? a / b : 0`

单局评分会计算以下指标：

| 指标 | 公式 | 含义 |
| --- | --- | --- |
| 伤害占比 | `个人英雄伤害 / 团队英雄伤害` | 输出份额 |
| 经济占比 | `个人经济 / 团队经济` | 资源占用 |
| 原始伤转 | `伤害占比 / 经济占比` | 拿多少经济打多少输出 |
| 修正伤转 | `伤害占比 / 修正经济占比` | 考虑人头经济后的伤转 |
| 人头占比 | `个人击杀 / 团队击杀` | 击杀份额 |
| 参团率 | `(击杀 + 助攻) / 团队击杀` | 团战参与 |
| 死亡占比 | `个人死亡 / 团队死亡` | 死亡负担 |
| 承伤占比 | `自我缓和伤害 / 团队自我缓和伤害` | 承伤份额 |
| 每死承伤 | `(个人自我缓和伤害 / max(死亡, 1)) / (团队自我缓和伤害 / max(团队死亡, 1))` | 承伤质量修正 |
| 治疗占比 | `个人治疗 / 团队治疗` | 治疗贡献 |
| 定身占比 | `个人定身次数 / 团队定身次数` | 控制量 |
| 定身击杀占比 | `个人定身后协助击杀 / 团队定身后协助击杀` | 控制结果 |
| 定身转化 | `个人定身后协助击杀 / 个人定身次数` | 控制质量 |

### 修正伤转

原始伤转的问题是：击杀会带来额外经济。如果一个玩家确实通过高输出拿到人头，经济高不应该完全惩罚他。

因此代码会计算一个修正经济占比：

```text
killQuality = clamp01(1 - max(0, 人头占比 - 伤害占比) / 0.18)
productiveKillShare = min(人头占比, 伤害占比 + 0.04)
killPressure = clamp01((productiveKillShare - 0.18) / 0.18)
killGoldRelief = 0.06 * killPressure * killQuality

修正经济占比 = max(
  经济占比 - killGoldRelief,
  经济占比 * 0.78,
  0.08
)

修正伤转 = 伤害占比 / 修正经济占比
```

简单说：

- 人头占比高但伤害占比低，会被认为偏 K 头，修正很少。
- 人头占比高且伤害占比也高，会适当降低经济分母，减少“打得好所以经济高”带来的伤转惩罚。

### 伤转死亡门控

修正伤转本身仍然是核心指标，但不会直接全额进分。为了避免“无脑送死导致经济占比低，从而刷出高伤转”的情况，代码会把伤转分再经过一层死亡门控：

```text
原始伤转分 = clamp01((修正伤转 - 基础线) / 成长区间)

死亡压力 = clamp01((死亡占比 - 0.20) / 0.12)
输出保护 = clamp01((伤害占比 - 0.24) / 0.12)
死亡伤害错位风险 = clamp01((死亡占比 - 伤害占比 - 0.02) / 0.12)

送死风险 = max(
  死亡压力 * (1 - 输出保护),
  死亡伤害错位风险 * (1 - 输出保护 * 0.4)
)

伤转保留比例 = clamp(1 - 送死风险 * 0.82, 0.18, 1)
最终伤转分 = 原始伤转分 * 伤转保留比例
```

简单说：

- 伤害占比高，说明这次死亡风险有输出结果，伤转分会被保护。
- 死亡占比高但伤害占比只是中低水平，伤转分会被明显压低。
- 最低仍保留 18% 的伤转分，避免一刀切误伤特殊局。

### 控制质量

控制不直接用控制时长，因为减速等廉价控制会污染这个指标。当前使用定身相关数据：

```text
控制质量 =
  clamp01(定身占比 / 0.28) * 0.35
  + clamp01(定身击杀占比 / 0.30) * 0.45
  + clamp01(定身转化 / 0.22) * 0.20
```

控制质量更看重“定身后产生击杀结果”，其次才是定身次数。

## 定位判断

定位判断位于 `roleClassifier.ts`。

基础定位有 6 类：

- `adc`
- `mage`
- `assassin`
- `fighter`
- `tank`
- `support`

最终显示定位还会派生出：

- `bruiser`：半肉战士
- `pureAssassin`：纯刺客
- `fighterAssassin`：战士刺客
- `utilityCarry`：功能输出
- `unknown`：输出

### 英雄定位

英雄资源中每个英雄有 `roles`。第一个英雄标签权重更高：

```text
主标签权重 = 1.25
副标签权重 = 0.85
```

随后归一化成英雄定位向量。

### 装备定位

装备定位来自三部分：

1. 装备图标路径，比如 `marksman`、`mage`、`assassin`、`fighter`、`tank`、`support`。
2. 装备分类和描述，比如暴击、攻速、穿甲、法强、生命值、护甲、治疗护盾强度等。
3. 手写装备修正表 `ITEM_FIXES`，用于修正游戏资源里分类不够准确的装备。

装备权重按价格计算：

```text
price <= 0      -> 0
price < 900     -> price * 0.20
price < 1800    -> price * 0.45
price >= 1800   -> min(price, 3600)
```

这样小件、鞋子、消耗品不会过度影响定位，成装会成为主要证据。

### 英雄和装备的合并

装备权重越高、装备倾向越集中，装备越能决定本局定位。

```text
itemDominance = 装备定位向量中的最大值

if itemGold >= 4500:
  itemRatio = itemDominance >= 0.75 ? 0.78 : 0.65
else if itemGold >= 2200:
  if itemDominance >= 0.80: itemRatio = 0.72
  else if itemDominance >= 0.65: itemRatio = 0.62
  else itemRatio = 0.52
else:
  itemRatio = itemDominance >= 0.85 ? 0.55 : 0.40

championRatio = 1 - itemRatio
最终定位向量 = 装备向量 * itemRatio + 英雄向量 * championRatio
```

### 最终定位规则

按顺序判断：

| 条件 | 定位 |
| --- | --- |
| `fighter >= 0.30 && tank >= 0.24` | 半肉战士 |
| `fighter >= 0.28 && assassin >= 0.24` | 战士刺客 |
| `assassin >= 0.46 && tank < 0.16 && fighter < 0.26` | 纯刺客 |
| `support >= 0.38 && max(adc, mage, assassin, fighter) < 0.42` | 辅助 |
| `support >= 0.28 && max(adc, mage) >= 0.28` | 功能输出 |
| `tank >= 0.42 && fighter < 0.24` | 坦克 |
| 最高权重定位 `>= 0.22` | 对应定位 |
| 其他 | 输出 |

## 评分分支

单局评分会先把定位映射到评分分支：

| 定位 | 评分分支 |
| --- | --- |
| 坦克 | 前排 |
| 战士、半肉战士、战士刺客 | 战士 |
| 辅助、功能输出 | 辅助 |
| ADC、法师、刺客、纯刺客、输出 | 输出 |

最终分数为各项子分之和，随后：

```text
最终得分 = round(clamp(子分总和, 0, 100))
```

## 输出分支

输出分支适用于 ADC、法师、刺客、纯刺客和未知输出。

输出分支使用一个“伤害、伤转、承伤动态分池”。法师分池略小，但法师保留更高参团权重。

### 动态战斗分池

法师：

```text
战斗分池 = 63
```

其他输出：

```text
战斗分池 = 69
```

参数：

```text
承伤最高可借权重 = 20
伤害权重比例 = 28 / 55
伤害目标 = 0.30
伤转基础线 = 0.78
伤转成长区间 = 0.57
承伤目标占比 = 0.22
承伤目标每死 = 1.25
承伤占比启动线 = 0.22
承伤占比成长区间 = 0.24
每死承伤启动线 = 0.90
每死承伤成长区间 = 0.65
承伤优势区间 = 0.45
```

动态分池公式见“动态承伤权重”一节。

输出分支额外子分：

| 子分 | 公式 |
| --- | --- |
| 参团 | 法师 `14 * clamp01((参团率 - 0.45) / 0.35)`；其他输出 `8 * clamp01((参团率 - 0.45) / 0.35)` |
| 人头质量 | `9 * 人头质量` |
| 生存 | `8 * (1 - clamp01((死亡占比 - 0.20) / 0.18))` |
| 经济效率 | `6 * 伤转死亡门控(0.75, 0.55)` |

战士分支适用于战士、半肉战士、战士刺客。

### 动态战斗分池

```text
战斗分池 = 79
承伤最高可借权重 = 22
伤害权重比例 = 36 / 58
伤害目标 = 0.28
伤转基础线 = 0.72
伤转成长区间 = 0.58
承伤目标占比 = 0.30
承伤目标每死 = 1.15
承伤占比启动线 = 0.18
承伤占比成长区间 = 0.22
每死承伤启动线 = 0.70
每死承伤成长区间 = 0.60
承伤优势区间 = 0.35
```

战士分支额外子分：

| 子分 | 公式 |
| --- | --- |
| 参团 | `8 * clamp01((参团率 - 0.43) / 0.37)` |
| 生存 | `8 * (1 - clamp01((死亡占比 - 0.24) / 0.20))` |
| 人头质量 | `5 * 人头质量` |

## 前排分支

前排分支适用于坦克。

| 子分 | 公式 | 满分 |
| --- | --- | --- |
| 承伤质量 | `30 * 门控承伤质量(0.34, 1.20)` | 30 |
| 有效伤害 | `17 * clamp01(伤害占比 / 0.24)` | 17 |
| 参团 | `15 * clamp01((参团率 - 0.42) / 0.38)` | 15 |
| 死亡控制 | `10 * (1 - clamp01((死亡占比 - 0.28) / 0.22))` | 10 |
| 定位匹配 | `8 * clamp01((坦克权重 + 战士权重) / 0.65)` | 8 |
| 效率 | `9 * 伤转死亡门控(0.65, 0.50)` | 9 |
| 治疗 | `5 * clamp01(治疗占比 / 0.25)` | 5 |
| 控制 | `6 * 控制质量` | 6 |

前排不会完全忽略输出。伤害占比和修正伤转仍然会给分，只是承伤质量占比最高。

## 辅助分支

辅助分支适用于辅助和功能输出。

```text
保护值 = max(治疗占比, 承伤占比 * 0.75)
```

| 子分 | 公式 | 满分 |
| --- | --- | --- |
| 参团 | `23 * clamp01((参团率 - 0.45) / 0.35)` | 23 |
| 治疗 | `18 * clamp01(治疗占比 / 0.32)` | 18 |
| 保护 | `14 * clamp01(保护值 / 0.30)` | 14 |
| 有效伤害 | `13 * clamp01(伤害占比 / 0.22)` | 13 |
| 死亡控制 | `9 * (1 - clamp01((死亡占比 - 0.24) / 0.20))` | 9 |
| 经济克制 | `6 * (1 - clamp01((经济占比 - 0.23) / 0.18))` | 6 |
| 人头质量 | `7 * 人头质量` | 7 |
| 控制 | `10 * 控制质量` | 10 |

## 动态承伤权重

输出和战士的承伤权重不是固定的。承伤可以为 0，也可以从“伤害和伤转分池”里借权重，但必须满足三个条件：

1. 承伤质量分高于非承伤战斗表现。
2. 承伤占比达到启动线。
3. 每死承伤达到启动线。

基础分：

```text
伤害分 = clamp01(伤害占比 / 伤害目标)
伤转分 = 伤转死亡门控(伤转基础线, 伤转成长区间)

承伤质量 =
  clamp01(承伤占比 / 承伤目标占比) * 0.65
  + clamp01((每死承伤 - 0.55) / (承伤目标每死 - 0.55)) * 0.35

非承伤战斗分 =
  伤害分 * 伤害权重比例
  + 伤转分 * (1 - 伤害权重比例)

承伤优势 = 承伤质量 - 非承伤战斗分
```

只有 `承伤优势 > 0` 时才会借权重：

```text
承伤占比门槛 = clamp01((承伤占比 - 承伤占比启动线) / 承伤占比成长区间)
每死承伤门槛 = clamp01((每死承伤 - 每死承伤启动线) / 每死承伤成长区间)
优势门槛 = clamp01(承伤优势 / 承伤优势区间)

boost = 承伤占比门槛 * 每死承伤门槛 * 优势门槛
承伤权重 = 承伤最高可借权重 * boost
```

剩余权重继续分给伤害和伤转：

```text
非承伤权重 = 战斗分池 - 承伤权重
伤害权重 = 非承伤权重 * 伤害权重比例
伤转权重 = 非承伤权重 - 伤害权重

动态伤害子分 = 伤害权重 * 伤害分
动态伤转子分 = 伤转权重 * 伤转分
动态承伤子分 = 承伤权重 * 承伤质量
```

这个设计的目的是：

- 输出位不会因为刷承伤获得高分。
- 战士承伤高且每死承伤质量好时，可以获得额外收益。
- 承伤收益来自伤害/伤转分池，不会凭空增加总权重。

## 人头质量

人头质量用于区分“确实在 C”和“只会 K 头”。

```text
K头差 = max(0, 人头占比 - 伤害占比)
人头质量 = clamp01(1 - K头差 / 0.18)
```

人头占比明显高于伤害占比时，人头质量下降。

## 前排承伤质量

前排使用更严格的门控承伤质量：

```text
承伤占比分 = clamp01(承伤占比 / 目标承伤占比)
每死承伤分 = clamp01((每死承伤 - 0.55) / (目标每死承伤 - 0.55))
门控承伤占比分 = 承伤占比分 * (0.35 + 每死承伤分 * 0.65)

门控承伤质量 = 门控承伤占比分 * 0.70 + 每死承伤分 * 0.30
```

也就是说，坦克无脑送死刷承伤不会稳定拿满分；每死承伤越低，承伤占比的价值越低。

| `>= 82` | excellent / 金色 |
| `65-81` | good / 绿色 |
| `41-64` | average / 蓝色 |
| `< 41` | poor / 红色 |

实时战绩和查看历史能力中也会用更口语化的能力段位：

| 分数 | 评价 |
| --- | --- |
| `>= 90` | 通天代 |
| `>= 82` | 小代 |

## 玩家画像

玩家画像由 `buildPlayerProfile` 生成。它只聚合调用方已经传入的对局，不在前端额外请求战绩。

输出字段：

| 字段 | 算法 |
| --- | --- |
| 综合分 | 所有单局得分平均值 |
| 中位数 | 所有单局得分中位数 |
| 高光率 | `得分 >= 80` 的对局占比 |
| 战犯率 | `得分 < 60` 的对局占比 |
| 波动 | 单局得分标准差 |
| 主玩定位 | 出现次数最多的定位 |
| 定位分布 | 每种定位的场次、胜场、占比、胜率 |
| 输出能力 | 输出类对局的能力画像 |
| 前排能力 | 前排类对局的能力画像 |
| 辅助能力 | 辅助类对局的能力画像 |

能力线映射：

| 单局定位 | 画像能力线 |
| --- | --- |
| 坦克、半肉战士 | 前排能力 |
| 辅助、功能输出 | 辅助能力 |
| 其他定位 | 输出能力 |

因此当前画像里，普通战士、战士刺客仍归入输出能力线；半肉战士归入前排能力线。

## 画像标签

玩家画像会先加入一个聚合评价标签，然后根据平均指标追加多个标签，最多保留 10 个。

### 聚合评价标签

聚合评价不是简单找所有标签里出现最多的，而是先找分段：

| 分段 | 条件 |
| --- | --- |
| 高分段 | `>= 80` |
| 正常段 | `60-79` |
| 低分段 | `40-59` |
| 战犯段 | `< 40` |

规则：

1. 先统计每个分段的场次。
2. `<40` 战犯段只有占比 `>= 30%` 时才有资格成为主分段。
3. 在有资格的分段里，选择场次最多的分段。
4. 在该分段内部，选择出现次数最多的单局标签作为画像主标签。

这样可以避免“少量纯战犯局”压过大量高分局，也避免高分局标签过于分散时被低分标签轻易盖住。

注意：这里的 `<40` 是“画像主标签分段保护”的战犯段；画像面板里的战犯率统计口径是 `得分 < 60`，两者不是同一个概念。

### 画像附加标签

| 条件 | 标签 |
| --- | --- |
| `平均伤害占比 >= 0.27 && 平均修正伤转 >= 1.15` | 输出爆表 |
| `平均伤害占比 < 0.20 && 平均修正伤转 < 0.95` | 低能输出 |
| `平均修正伤转 >= 1.18` | 伤转优秀 |
| `平均人头占比 - 平均伤害占比 >= 0.08 && 平均伤害占比 < 0.24` | 只会K头 |
| `平均参团率 < 0.52 && 平均伤害占比 < 0.23` | 开游戏的 |
| `平均参团率 >= 0.72` | 团战积极 |
| `高光率 >= 0.28` | 高光多 |
| `战犯率 >= 0.25` 或 `战犯率 >= 0.18 && 综合分 < 65` | 战犯偏多 |
| `战犯率 > 0 && 综合分 >= 70` | 偶有拉胯 |
| `波动 <= 9 && 样本 >= 8` | 发挥稳定 |

## 单英雄画像

单英雄画像由 `buildChampionProfiles` 生成：

1. 按英雄 ID 分组。
2. 对每个英雄的所有对局重新计算评分。
3. 统计该英雄的主定位、聚合标签、高光率、战犯率、平均伤害占比、平均伤转、平均承伤占比、平均治疗占比。

其中单英雄战犯率同样按 `得分 < 60` 统计。

单英雄平均分不是简单平均，而是加权分：

```text
单英雄评分 =
  平均分 * 0.60
  + 中位数 * 0.25
  + 主分段平均分 * 0.15
```

其中“主分段”的选择和玩家画像一致：`>=80`、`60-79`、`40-59`、`<40`，且 `<40` 分段只有占比达到 `30%` 时才有资格成为主分段。

## 当前设计倾向

当前评分系统有几个明确倾向：

1. 输出位以伤害占比和修正伤转为核心，不再给 KDA 独立大权重。
2. 人头占比必须结合伤害占比看，避免“只会 K 头”被高估。
3. 承伤必须结合每死承伤看，避免无脑送死刷承伤。
4. 输出和战士的承伤权重是动态的，承伤表现只有优于伤害/伤转表现时才会借权重。
5. 坦克、辅助有独立公式，但仍保留一定输出、效率或控制权重。
6. 玩家画像先按分段聚合，再找分段内主标签，避免少量低分局轻易覆盖大量高分局。

## 可调参数入口

常见调整点：

- `scoring.ts`
  - 各分支权重。
  - 动态承伤分池参数。
  - 单局标签阈值。
  - 分数色阶。
- `roleClassifier.ts`
  - 装备修正表 `ITEM_FIXES`。
  - 装备和英雄定位合并比例。
  - 最终定位判定阈值。
- `playerProfile.ts`
  - 主分段保护阈值 `DISASTER_BAND_DOMINANT_RATE`。
  - 战犯率分数线 `DISASTER_RATE_SCORE_LIMIT`。
  - 玩家画像标签阈值。
  - 单英雄加权平均公式。
